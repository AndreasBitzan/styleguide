Coding Conventions

  • The relative unit rem is preferred! If not applicable, use px.
  • We use indentations inside each tag to show the relationship of each tag.
  • We use SASS as our preprocessor, so that we would be able to use variables.
  • Each page has its own .scss file to go along with it, when necessary.
  • There is a variable.scss file to store our consistent styling, such as colors and fonts.
  • Within our CSS, we first style fonts, then colors, and then spacing is always last.
  • We always put each element of spacing in a single line. Such as margin-top vs. margin-bottom.
  • Javascript lines have to end with a semicolon ;

SCSS Conventions

Marging and padding is not allowed to be written in one line.

                
    padding: 1rem 7rem 5rem 7rem; //NO

    padding-top: 1rem;  //YES
    padding-right: 7rem;
    padding-bottom: 5rem; 
    padding-left: 7rem;

Curly braces { always open up directly beneath the tag/classname/id.

                
    h4
    {
    } //NO

    h4{
    } //YES

Styling properties that do not directly belong to each other, must be seperated by an empty line.

                    
    padding-bottom: 8px;
    padding-left: 0;
    margin-top: 0;
    margin-right: 1rem;  //NO

    padding-bottom: 8px;
    padding-left: 0;

    margin-top: 0;
    margin-right: 1rem; //YES

SCSS Property Order

The SCSS files must be designed in the correct order:

  1. Positioning (absolute, relative, top, bottom, etc.)
  2. Display properties (flex, block, grid, etc.)
  3. Font properties (font-size, font-family, etc.)
  4. Colors (color, background-color, etc.)
  5. Margin and Padding are always the last properties!
